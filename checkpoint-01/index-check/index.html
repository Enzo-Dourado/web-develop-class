Instruções

  1. Crie um arquivo participantes.md com o nome completo de todos os alunos participantes.
  2. Crie um repositório privado no seu GitHub com o nome checkpoint-1.
  3. Vá em Settings > Manage access (ou Collaborators) e adicione o e-mail cintra.70@gmail.com como colaborador, para que o professor tenha acesso.
  4. Se o trabalho for realizado em dupla ou trio, o repositório deve ser criado por apenas uma pessoa, que adicionará os colegas como colaboradores.
    Exemplo: Lucas cria o repositório e adiciona Mateus como colaborador e outros colegas.
  5. Clone o repositório na sua máquina local.
  6. Crie 10 pastas (uma para cada questão): questao1, questao2, ..., questao10.
    • Exemplo de estrutura:
      • checkpoint-ambiente-git-js/
      •   questao1/
      •   questao2/
      •   questao3/
      •   ...
      •   questao10/
  7. Faça commits separados, um para cada questão, com mensagens de commit claras e padronizadas (ex.: "Questão 1 – Versões", "Questão 2 – HTML + JS", etc.).
  8. Faça push das alterações para o GitHub.
  9. Envie o link do repositório privado como resposta, garantindo que o professor já tenha acesso e consiga visualizar o repositório.
